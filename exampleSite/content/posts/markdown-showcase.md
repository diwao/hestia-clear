This post includes common Markdown elements so the typography plugin, spacing, borders, and inline styles can be reviewed in one place.

## Headings

Headings should create clear hierarchy without overwhelming the article body. A second-level heading is useful for major sections, while third-level headings can group smaller notes.

### A Smaller Heading

Use smaller headings for supporting ideas, not for every paragraph.

## Lists

Ordered lists should be easy to scan:

1. Define the goal.
2. Write the smallest useful draft.
3. Review the page on desktop and mobile.

Unordered lists should feel compact:

- Keep labels short.
- Prefer concrete examples.
- Remove anything that does not help the reader.

## Inline Content

Inline code such as `hugo server` should remain legible inside a paragraph. Strong text should be useful for emphasis, and italic text should not disturb the rhythm of the line.
